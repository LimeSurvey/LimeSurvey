安裝

1. 請確定您的網站能運行 LimeSurvey .
您的網站伺服器需要 PHP 4.1.0 以上版本及 MySQL 數據庫. 而且須預留 2 至 2.5 MB 磁碟空間.

2. 下載 the LimeSurvey 套件
到訪 sourceforge 網站下載最新版 LimeSurvey 套件. http://sourceforge.net/projects/limesurvey/
套件約 300kb 至 400kb 的檔案大小. 下載 zip 壓縮檔，並儲存到您的電腦硬碟內.

3. 反壓縮 LimeSurvey 套件
使用壓縮程式把 zip 壓縮檔案還原到您的目錄／資料夾. 反壓縮時請保持原來的目錄結構及檔案配置一併進行 (這是大多數壓縮程式的預設處理方法). 反壓縮的套件具備下列結構:

目錄使用權限
程序 admin 目錄需要有 '寫入' 權限來輸入外部問卷. "/limesurvey/tmp" 目錄用作上傳檔案，所以要設定 可讀取/可寫入/可執行全部的狀態. 在 linux 或 unix 它應該 chmod 到 777. 如果您希望使用 LimeSurvey 來控制 Apache 目錄的安全狀態, /phpsurveyor/admin 目錄要設定成可讀取/可寫入/可執行全部的狀態; 在 linux/unix chmod 777. 如果您計劃使用其它方法來設定 Apache 目錄的安全設定 (ie: 手動或 使用網絡管理工具) 您可以讓此目錄設定成 可讀取/可執行狀態 - chmod 755. 其他目錄可以設定成可讀取狀態或在 linux/unix chmod 755. 您可以把 /limesurvey/admin 目錄內的檔案都設定成 755 的使用權限.

4. 收集網站伺服器資料
您需要符合以下條件才可以安裝 LimeSurvey.

  • 您要建立 limesurvey 的網址(ie: 'http://my.domain.com/limesurvey')

  • limesurvey 在伺服器所在的實質位置 (ie: '/home/usr/htdocs/limesurvey')

  • 您的 MySQL 數據庫的 IP/網絡位置 (ie:'localhost')

  • 如果您的 MySQL 數據庫使用非標準接口, 您就要找出接口編號.

  • 您的 MySQL 數據庫使用的用戶名稱及密碼

  • 如果您的網絡伺服器是使用 Windows 操作系統, 請當試找出 mysql bin 檔案的存放位置. (ie: 'c:/mysql/bin')

5. LimeSurvey 組態
使用文字編輯器修改 /limesurvey/admin 目錄內的 config.php 檔案. 下列設定要加以調整:

  • $databaselocation: 把 MySQL 數據庫設定成 IP/網絡位置. 大多數情況下設定為 "localhost" 即可.

  • $databaseport: 使用 MySQL 數據庫伺服器設定的接口. 大多數情況下設定成 "3306" 即可 (這是 MySQL 預設).

  • $databasename: 在 MySQL 數據庫伺服器內使用 LimeSurvey 的數據庫名稱 . 如果您在 MySQL 伺服器 擁有高度的使用權限, 您可以為LimeSurvey 建立專用數據庫. 在此情況下，在此設定填上其名稱. 此外，您亦可以使用原有的MySQL 數據庫名稱.

  • $databaseuser: MySQL 用戶名稱

  • $databasepass: MySQL 密碼

  • $databasetabletype: 建立新資料表時預設的資料表類型 - "MyISAM"
     

  • $useidprefix: 預設是 0 (zero). 如果定為 1 (one) 問卷的回應結果會採用預設的前綴 id 編號, 由以下字串設定 $idprefix. 您可以使用這方法來為同一份問卷建立多個版本的 LimeSurvey ， 然後把不同伺服器的問卷結果匯總成單一回應結果的資料表.

  • $idprefix: 前綴為伺服器每個問卷回應結果加上特有的 id 編號.
     

  • $rooturl: 執行 limesurvey 程序檔的正確 URL 網址. 其實您毋須在此作出設定，因為預設變數已為您作好安排 ( {$_SERVER['SERVER_NAME']} 會自動找出執行檔的所在，萬一不可，您才需要在此設定 URL - 例如︰ "http://www.mydomain.com/limesurvey".

  • $rootdir: 執行 limesurvey 程序檔的絕對位置. 其實您毋須在此作出設定，因為預設變數已為您作好安排 ( {$_SERVER['DOCUMENT_ROOT']} 會自動找出執行檔的所在，萬一不可，您才需要在此設定 - 例如︰ "/home/public_html/limesurvey".

    • 以下的位置設定已是 LimeSurvey 的標準設定，如無必要切勿修改 (ie: 只要您是使用標準的目錄結構) ，除非您為子目錄改了名，才需要在此設定.

    • $homeurl: 這裡要設定admin 子目錄的 URL . 這是 /limesurvey/admin 資料夾的網址 URL - 例如︰ "http://www.mydomain.com/limesurvey/html/admin". 結尾切勿加上斜線. Config.php 的預設設定會自動找尋伺服器名稱內的php 變數設定 - {$_SERVER['SERVER_NAME']}. 一般而言，您只要保留本欄設定，只需改LimeSurvey 所在目錄的名稱以便相互匹配.

    • $publicurl: 這是伺服器'公用主目錄' 指向的 URL . 所有公用程序包括 "limesurvey" 資料夾(或其他您安放其他程序`的子目錄) 均存放在公用主目錄之下.

    • $tempurl: 這是設定 "/limesurvey/tmp" 目錄的 URL 位置 - 或您想把 LimeSurveyor 存放臨時檔案及上傳檔案的目錄位置. 這目錄必須具有 讀/寫/執行的權限 (ie: chmod 777)

    • $imagefiles: 預設是指向 /limesurvey/admin/images 的 URL 位置 - 圖片會自動存放於此。如果圖片是存放在其他位置，請您在此設定其 URL 正確位置.

    • $homedir: 指向 admin 執行檔的絕對位置 - 例如︰"/home/usr/htdocs/limesurvey/admin". 結尾切勿加上斜線. 在 config.php 預設值會採用 php 變數設定自動找尋全部文件的預設根目錄 - {$_SERVER['DOCUMENT_ROOT']}. 在大多數情況下您只需要把 LimeSurvey 程序檔所在的目錄名稱修改匹配即可。

    • $publicdir: 在此設定 '公用程序檔' 目錄的絕對位置.

    • $tempdir: 在此設定 /limesurvey/tmp 目錄的絕對位置，讓程序可以讀寫檔案.
       

  • $sitename: PHPsurveyor admin.php 程序檔執行後在網頁上顯示的名稱. (ie: "本機構的問卷工具")

  • $scriptname: 除非您已更改 admin.php 的名稱，否則讓本欄預設為 "admin.php". 如果您想在網址列以標題 "index.php"方式執行, 您就要把admin.php 檔改名, 並在此設定為 "index.php".

  • $accesscontrol: 此設定決定 LimeSurvey 是否要做安全設定. "1" 的設定值開啟本功能, "0" 的設定值關閉本功能. 即使關閉本功能設定，您仍可透過伺服器供應商的內建安全密碼去保護執行目錄，只不過安全保護工作不再由 LimeSurvey 偵測及處理罷了。如果您並非使用 Apache 網站伺服器，只要把本欄設定成 "0" 即可.

  • $defaultuser: 如果啟用了本系統的安全設定，在此就是預設的用戶，去通過 LimeSurvey 建立的安全設定.

  • $defaultpass: 如果啟用了本系統的安全設定，在此就是預設的密碼，去通過 LimeSurvey 建立的安全設定.

  • $siteadminemail: 網管的電郵地址

  • $siteadminname: 網管名稱

  • $surveyfaxnumber: '列印版本'問卷需要用到的傳真號碼.
     

  • $dropdowns: 本欄可設定成 "L" 或 "R". 設定成 "R" ，會以小圓孔按鈕作為列表的題型, 而 "L" 會以下拉式框架作為列表的題型.

  • $lwcdropdowns: 與前者作用相同，但 '列表附有評語欄位' 的題型.

  • $dropdownthreshold: 當您為 $dropdowns 選擇 "R" 設定, 讓您以以小圓孔按鈕的形式設定最大數目的選項，然後以下拉式列表顯示. 如果您有題目需要用到相當多的選項,如果以小圓孔的按鈕形式顯示，看來比較順眼. 最大設定值 25 (預設值)讓問卷參與者更易使用.

  • $repeatheaders: 以矩陣形式的題型(自設標題)表示您有很多答題顯示在螢幕上，長度足以用上超過一個頁面. 先決定了答案的數目，才重覆題目的標題資訊. 比較常用是 15 的設定值. 如果您不需要重覆標題, 可設定為 0 (預設值).
     

  • $maxemails: 當要寄出邀請或提示電郵給問卷填寫人, 本設定可決定一次過發出多少封電郵. 不同的網站伺服器有不同的電郵處理容量, 如果寄發電郵程序執行需時，會造成 time out 及出錯. 絕大部次網站伺服器可以在預設 30 秒時間內寄出 100 封電郵. 若有 time out 及出錯，嘗試降低此設定值.

  • $defaultlang: 本欄會設定 admin.php 執行畫面及公用問卷的預設語言顯示. 您可以更改本設定，令 admin.php 執行公用問卷時按逐份問卷的需要作出改變。

  • $OS: 本項毋須設定，因為它會偵測網站伺服器的操作系統.

  • $apachedir: 本設定告訴 LimeSurvey 找出 apache 執行檔的位置，但在 Linux/Unix 伺服器，本欄位可以留空(?;) LimeSurvey 會自動查找位置. Windows 伺服器可能需要填寫本欄位. (eg: $apachedir = ?a href="file:///c:/program">c:/programfiles/apachegroup/apache/bin?)

  • $mysqldir: 本設定告訴 LimeSurvey mysql 二元碼檔案的位置 (特別是 Mysqldump?. 如果是 Linux/Unix 伺服器，可以不用切定本欄位(?;) LimeSurvey 會自動查找. 但 Windows 伺服器就需要填寫本欄位. (eg:$mysqldir = :/mysql/bin?)

  • $timeadjust: 如果用戶所在地與網站伺服器所處的時區不同，就要在時差設定反映。例如︰ 我住在澳洲但使用位於美國的網站伺服器. 後者較您所在地慢 14 小時在您的時差設定為 "14". 換言之，在網站伺服器要加上14 小時。由於完成網上問卷後會加上時間記錄，所以此設定十分重要。

  • $dbprefix: 如果您使用另一套獨立的 PHPsurveyor 數據庫，請留空本欄 (ie:  $dbprefix="";) 如果您想在 LimeSurvey 及其他數據型軟件作出共享的設定，就需要在本設定加上前綴。我建議的前綴是 "phpsv_", 但名稱可以悉隨尊便去更改.

  • $allowmandbackwards: 本設定為 1 表示填寫問卷的人士即使未回答必答題，仍可以返回上一道題目繼續作答；若設定為 0, 表示填寫問卷的人士在返回上一題目前，必須先行回答現時的必答題.

  • $deletenonvalues: 這項設定很難向您解釋清楚. 若設定為 0, 而當中一道題目 (或許在此稱之為 "b" 題目) 有條件地顯示 (ie: 它的出現是建基於上一道題目的答案滿足特定條件 - 在此稱之為 "a" 題目), 當答題者回應了 "b" 題目後, 但他後悔又想返回上一道的 "a" 題目重新改答案，如此令到 "b"題目 不能再出現 (因為答案改變了, 令 "b"題目出現的條件不復存在), 本設定令問卷仍會記住原來出現 "b" 題目您曾回答過的答案，把它記錄下來. 我覺得本功能有可取之處，所以提供大家多一個選擇。但如果您寧可保持問卷操作的完整性，而認為不應該把不合理的資料儲存下來，您可以把本欄設定為 1, 問卷系統就不會額外儲存上述的資訊了。

  • $shownoanswer: 當採用小圓孔按鈕／選擇按鈕的題目包含可編修的答案(ie: 列表、陣列式題目) 並非必答題，而本欄 $shownoanswer 的設定為 1, 題目會額外出現 "N/A" 的欄位- 表示答題者可以選擇不回應本題題目. 但有些問卷設計人並不喜歡這樣的設定，如此您只需要把本欄設定為 0 就可以關閉本功能.
     

  • $usejpgraph: 如果您有為伺服器正確設定 jpgraph, 您可以開啟本功能 (1=開, 0=關) ，這樣統計程式會把問卷摘要結果以批形圖顯示，但本功能在開發上仍未臻善美，運作上仍有不足之處.

  • $jpgraphdir: jpgraph 程式的絕對位置. 本設定必須配合 $usejpgraph 是 1 的設定值.
     

  • $btstyle: 您可以不理會本設定，它的功能僅為 admin.php 設序提供預設 CSS 按鈕風格的設定.

  • $slstyle: 您可以不理會本設定， 它的功能題為文字輸入框設定預設的 CSS風格.

6. 上傳檔案到您的網站伺服器.
採用您的 FTP 程式連結到您的網站伺服器，建立目錄存放您的程式. 然後依照它們原有結柟上傳檔案.

7. 首次連結 admin 程序
上傳檔案後，您可以透過網站瀏覽器建立 LimeSurvey. 打開您的瀏覽器，在網址列鍵入 admin.php 的 url 網址. 如果您是使用limesurvey 作為目錄名稱來存放它的所有檔案, 網址大概的形式如下 "http://your.domain.com/limesurvey/admin/admin.php"

當您首次使用 LimeSurvey, 程序會連結到 config.php 檔案預先設定好的數據庫. 如果相應的資料表結構未建立，LimeSurvey 會向您提供按鈕去建立所需要的數據庫資料表. 如果您建立資料表遇到問題，您可能需要執行類似 phpmyadmin 之類的數據庫管理程式去執行套件內的 "maketables.sql". 一旦建立此等資料表，您就可以操作 LimeSurvey 了.

8.若果我遇到問題，怎麼辦?
正如所有電腦程式, 運作過程難免偶有問題, 導致運作失常的原因也很多. 如果您操作上有麻煩, 請透過 sourceforge.net 的LimeSurvey 支援論壇發表您遇到的技術問題或 LimeSurvey 執行後出現的錯誤訊息，一定有人協助您解決困難。

 

安裝 FAQ

如果我的伺服器是開啟了 "safe_mode" (安全模式)， 怎麼辦?
如果您的伺服器開啟了 safe_mode (安全模式) (這是需要謹慎處理的安全設定) 您就不能使用 LimeSurvey 的部份內建功能。這包括建立和管理 Apache 伺服器的安全設定, 以及把 MySQL 的問卷結果輸出的能力。雖然這些功能與 LimeSurvey 賣點風馬牛不相及 - 但畢竟此等功能對管理者十分便捷. 您可以手動方式建立 Apache 的安全設定，您亦可以使用類似 phpMyAdmin 的數據庫程式去把 MySQL 的問卷結果輸出作進一步處理.

如果我無法透過安裝程序建立數據庫，怎麼辦?
一般而言, 安裝程序會自動建立全新的數據庫及資料表結構，但若果您的伺服器/ISP 不給與您管理 MySQL 數據庫的權限去建立資料表，您就需要向伺服器供應商聯絡，確保它們為您建立所需的數據庫. 它們或許會為您建立空白的數據庫，然後讓您有存取的權限 (ie: 讀取／寫入／建立)). 輸入數據庫的名稱，並在 config.php 檔案內設定用戶名稱及密碼, 然後就可以執行 admin.php 程序. 安裝過程需要逐次在數據庫建立每一筆所需的資料表。

我應該設定甚麼權限?
 /limesurvey/tmp 目錄需要設定 read/write/execute 的使用權限 (chmod 777) - 本目錄專鬥存放所有上傳檔案及暫時建立的檔案. 其他全部目錄及檔案可以設定為 read/execute 的使用權限即可 (chmod 755). 但是 如果您想使用內建的 apache 安全工具 (去建立 htpasswd 及 htaccess 檔案) admin 目錄就需要設定成 read/write/execute (777). admin 目錄裏面的檔案則可以設定成 read/execute (chmod 755).

安全問題

LimeSurvey 倚賴 Apache 的網站伺服器的 目錄 安全系統. 此外就沒有其他的安全保障了. 本安全設定是足夠應付所需, 但是如果安全考慮是您極度要關注的問題， 您就需要對網站的 Apache 安全管理有全盤的認知. 本軟件的作者對於此軟件的使用及相關的安全問題，不負任何責任. 換言之，除非您採取恰當的安全措施，您不能把本軟件視為理所當然的絕對安全.

如果您沒有使用 Apache (ie 您使用 Microsoft IIS) 您就需要在 config.php 檔案內把本安全設定關閉 ($accesscontrol = 0) , 及改用 IIS 的安全設定.

Linux 檔案的使用權限
如果您正在使用 Linux 伺服器，您就應該使用下列的檔案權限.

  • 對您的 limesurvey 目錄 - chmod 755 (使用 read, execute 權限去存取整個目錄)

  • 對於 tmp 目錄 - chmod 777 - 這目錄建成後是沒有檔案的，它是用來存放上傳的檔案、jpg 圖片及模組編輯器產生的暫存風格檔案

  • 對於 admin 目錄 - chmod 777 (讀取、寫入、執行的權限去存取全部檔案) 或 chmod 755 (讀取/執行)
    本資料夾需要具備寫入的權限，才可以產生 .htpasswd 和 .htaccess 檔案. 如果您準備用手動方式把本目錄做安全設定，您可以設定本目錄為 755

  • 對於 admin 目錄內的檔案 - chmod 755 (讀取、執行權限去存取目錄內的檔案)
    本資料夾的標準檔案毋須具備寫入的權限- 而且也無必要這樣做.

視窗檔案的使用權限

其他安全問題
config.php 檔案包含用戶名稱及密碼去存取您的 MySQL 資料夾. 這存在安全風險, 特別是您正在使用的登入方式對　MySQL 擁有極高的管理權限. 為了把安全風險降到最低, 您應該使用 LimeSurvey 程式的安全密碼的管理功能去保護 admin　網管目錄. 此外，為進一步降低風險，您亦可建立另一數據庫來存放問卷資料，使登入時只有存取此　limesurvey 數據庫的權限.

(電郵到 jason-at-cleeland.org). LimeSurvey 未來的版本可能會簡化　config.php 檔的處理程序.

問卷設計及結構

LimeSurvey 問卷有三個必要的組成部件. 每份問卷必須有:

  • 一個問卷名稱

  • 至少一個題目分組

  • 至少一道題目

問卷自選欄位:

  • 相關題目的適用答案

  • 可修改標籤

  • 決定題目應否顯示的條件句

問卷名稱 是一目瞭然的. 相關的設定包括歡迎信息、問卷說明及問卷管理員的聯絡信息以及問卷格式等等.

每個問卷把題目劃分群組. 這表示問卷依邏輯分割成群組分門別類. 每個群組可以擁有標題及說明. 即使您不想把問卷分割群組，在問卷中亦至少要有一個群組.

題目 是問卷的核心並可以群組的形式分門別類. 在技術層面問卷的題數或群組內的題數並無限制. 您可以設定題目內容、設定題型，亦可以為題目加入簡單的說明，並決定題目是否必須強制作答? (ie: 強制作答表示題目必須回答後才可以繼續填寫問卷).

建立問卷

  1. 建立新問卷

    要建立新問卷，請 click 選  管理按鈕列的右方按鈕.

    "建立新問卷" 的螢幕會顯示在下方. 下列是每一欄位的文字說明:

    1. 標題: 這是問卷名稱的扼要說明 (ie: "2003 年企業購買力問卷", 或 "冰其淋的觀點"). 本標題會在公開的問卷程序的每個頁面上顯示.

    2. 文字說明: 讓您輸入問卷的文字說明. (ie: "問卷用來收集下一輪企業購買力的意見" 或 "問卷找出朱古力冰淇淋受歡迎的程度"). 您可以在本章節使用 html markup 語法.

    3. 歡迎: 讓您可以輸入訊息，讓答卷人登入公開問卷時看到歡迎的訊息. (ie: "多謝您撥冗填寫本問卷..") 您可以在本章節使用 html markup 語法.

    4. 管理員: 負責管理本問卷的聯絡人的名稱. 它會在發出問卷邀請信的電郵內出現.

    5. 網管的電郵: 這是d網管的電郵地址 (見上) 及 '回覆人:' 的電郵地址，在電郵送出後被回覆所採用的電郵地址.

    6. 傳真到: 本欄是用於"印刷版本的傳統問卷"上顯示的傳真號碼 - ie: 當人們無法使用網上問卷，這時候這個印刷版本的問卷就用得著了。

    7. 格式: 選擇 "逐次處理", "逐個群組處理" 或 "一次過處理".

      • 逐次處理 公用問卷會在每頁顯示一道題目.

      • 逐批處理 公用問卷會在每頁顯示一個群組內的全部題目.
        逐個群組處理的問卷會把 "歡迎" 頁面和"送出" 頁面分開, 方式與 "逐次處理" 的問卷相同.

      • 一次過處理 的公用問卷會在單一頁面顯示全部題目
        一次過處理的問卷不會有 "歡迎" 頁面或 "送出" 兩個頁面 - 歡迎訊息和送出按鈕會在同一頁面出現.

    8. 模組風格: 在系統選擇已安裝的模組風格. 預設的模組風格是呆板但實用的. 更多有關建立自己風格模組的資訊詳見於 LimeSurvey 風格模組手冊

    9. 要使用 Cookies 嗎?: 如果您選擇 "使用 Cookies" ， 而您的問卷又不使用答卷代碼資料表去控制用戶的使用權限, 當答卷人送出問卷後， cookie 會儲存每個問卷的用戶電腦的登入資料. 這樣 Cookie 會防止此用戶採用同一部電腦去重覆填寫同一份問卷. 但'安全' 維護的角度，仍有不足之處。但大體而言，此功能對公用問卷就防止重覆填寫問卷的問題上，亦有幫助.

    10. 要儲存嗎?: 如果您要設定本項目為"是" ，那麼用戶可以把未提交的問卷暫時儲存，以便稍後回來繼續完成問卷.

    11. 顯示 [<< Prev] 按鈕: 本選項預設成 '是', 但若果您把它設定成 "否" ，則用戶在填寫問卷時，就不能返回上一頁面重新填寫了。

    12. 電郵通知: 當有人儲存問卷時， 此選項讓管理員可以收到電郵通知。您可以由此選擇:

      • 沒有電郵通知 - 毋須解釋

      • 基本電郵通知 - 電郵送出以通知管理員有問卷已被儲存

      • 送出電郵通知，並附有問卷回應的編號 - 儲存後送出問卷的完整答案

    13. 匿名: 讓您決定問卷是要答卷人引用答卷代碼才可以填寫問卷， 還是保持 '匿名'的方式. 預設是 '是'，表示您啟用問卷後，您一定要建立答卷代碼資料表才可以運行問卷； 如果您選擇 "否"，表示使用匿名問卷。

    14. 邀請電郵的主題: 當問卷使用答卷代碼時，邀請電郵發出時顯示的標題信息.

    15. 邀請電郵: 這是邀請電郵的文字說明，配合問卷使用答卷代碼時送出電郵的文字說明。 本欄會由預設的邀請訊息自動填寫的預設欄位(根據語言檔)，但您亦可以依照自己的需要加以修改。當然如果您無意在問卷使用答卷代碼, 本欄位就毋須設定了。
      您可以使用下列 "表格" 欄位，在每個電郵插入個人化的信息:

      • {FIRSTNAME} - 由答卷代碼表的 "姓氏" 欄位值所取代

      • {LASTNAME} - 由答卷代碼表的 "名" 欄位值所取代

      • {SURVEYNAME} - 由問卷名稱所取代

      • {SURVEYDESCRIPTION} - 由問卷的文字說明所取代

      • {ATTRIBUTE_1} - 由答卷代碼表的"自設欄位一" 欄位值所取代

      • {ATTRIBUTE_2} - 由答卷代碼表的"自設欄位二" 欄位值所取代

      • {SURVEYURL} - 由問卷的完整網址 URL所取代

      注意︰這些 "表格欄位" 適用於下列電郵地址的欄位.

    16. 電郵的提醒信的主題: 依照答卷代碼工具，送出答卷提醒電郵的主題名稱

    17. 電郵提醒信: 使用問卷答卷代碼時，提醒電郵發出時的電郵內容，詳見 "邀請電郵" 有關使用本欄位的細節說明.

    18. 確認電郵主題: 使用答卷代碼時, 這是問卷完成後，自動發送給答卷人的電郵主題名稱

    19. 確認電郵: 這是完成問卷後，發送後用戶的電郵內容.

    20. 容許公眾用戶註冊使用問卷系統: 如果您使用答卷代碼去操作問卷, 只有答卷代碼的人士才可以填寫相關問卷。如果您想用戶使用操作代碼，並容許公眾人士註冊後填寫問卷，請在本闌設定 "是"，讓訪客可以註冊用戶名稱及電郵地址，這樣系統就會在答卷代碼資料表建立新的答卷代扊，並透過邀請電郵寄送給註冊的用戶。請注意每個電郵地址只能註冊一個用戶，去填寫問卷。

    21. 公開註冊電郵的標題: 這是邀請電郵的標題，會發送給註冊用戶，邀請他們填寫問卷。

    22. 公開註冊電郵︰ 這是邀請註冊用戶填寫問卷的電郵的文本內容。 同樣 "表格欄位" 亦適用於本電郵。

    23. 答卷代碼自定欄位的名稱: 答卷代碼資料表有兩個"空"欄位來存放用戶的額外資料。當問卷使用公眾註冊形態時, 您可以額外增加此兩個自定欄位供用戶註冊時填寫。因此您若是使用 "自定欄位一" 去存放註冊用戶的部問名稱, 您就可以在自定欄位一賦予此相關的文字標籤了。

    24. 要使用日期紀錄嗎?: 本欄讓您決定問卷是否使用期紀錄全部問卷結果。若選 "是", 當問卷送出後，問卷結果就會新增欄位顯示當時填寫問卷的日期及時間. (有關 $timeadjust 設定，詳見組態設定.)

    25. 語言: 一系列語言檔案會以列表形式顯示，在此您可以更改公用問卷預設的語言顯示 (但不會改變管理選單的語言顯示的設定).

    26. 終止: 這是讓用戶填寫問卷的最後日期 (請仔細閱讀... 如果您設定成12月31日, 這表示到了翌年的1月1日人們就不能再填寫問卷了。

    27. 完成問卷後轉往新的 URL: 這個 URL 會在完成問卷後網頁自動連結的新網址, 例如把用戶帶往網站的主頁 (或其他網頁).

    28. URL 文字說明: 完成問卷轉往新連結網址 URL 的文字說明.

    29. 問卷完成後自動載入 URL: 如果您設定 "是"， 在問卷完成後，只會根據 "結束 URL" 的網址自動載入 URL，而並非顯示往常的 "已完成" 頁面. 如果您要為問卷使用評估工具，切勿啟用本設定

    Question Attributes are an extensible way of adding extra controls to the presentation and/or workings of questions. Additional question attributes can be developed relatively easily by PHP programmers and added to the list of options for a question.
    Different question types have different possible question attributes. Each attribute can have a "value" attached to it. This value may indicate whether the attribute is applied or not (usually a "1" to apply), or it may indicate a value used by the attribute.

    • display_columns:
      This attribute is available for radio button list and multiple choice type questions. It allows you to display the answers for the question in more than 1 column. Add this attribute, and a value indicating the number of columns you wish the question to use.
       

    • hide_tip
      This attribute allows you to turn off the "tip" for certain question types. For example, List type questions will usually include a tip that says "Please choose one of the following". Adding the "hide_tip" attribute with any value will turn this tip off so that it will not display.
       

    • random_order
      This attribute applies to any question type that has a list of possible answers. It causes LimeSurvey to ignore the existing order of those answers, and instead it orders the answers randomly. It is important to note that this random order will be reordered each time the question is displayed, so if someone returns to a question, the order will probably be different from the first time they saw it. Having the answers display randomly is useful to remove the possibility of bias from answers to questions.
       

    • code_filter
      Can be used to filter the answers displaying based on the answer to a previous question. The attribute was added for the purpose of being able to narrow down possible options.
       

    • permission
      Currently unused attribute that will eventually be integrated with the public display of results, so you can indicate whether a questions results are public or not
       

    • maximum_chars
      Available for text based questions. Allows you to set the maximum number of characters that can be entered for a text based question. Entering a value of, say, 20 will mean that the participant cannot enter any more than 20 characters as their answer.
       

    • text_input_width
      Available for short text and numerical text type questions. Sets the width of the html text box, but does not limit the number of characters that can be entered.

本土化

公用問卷畫面可以本土化，令到內建的訊息及文字可以透過您熟悉的語言展示出來。 例如︰ "是" 及 "否" 在德文 版本會是以 "ja" 及 "nein" 顯示。因此，LimeSurvey 提供相當多的語言檔案，然而您亦在 /limesurvey/lang 目錄建立自用的語言檔案，或者使用具備 UTF-8 編碼功能的編輯器加以修改，來切合自己的需要。 

建立 UTF-8 檔案，有一個小竅門: 要維持 UTF-8 語言檔的向後兼容性， UTF-8 檔案不可以包含 BOM (byte order mark). BOMs 是由兩個位元組組成，位於 UTF-8 的檔頭，用來描述正確的編碼. 我們建議使用以下兩個 編輯器，就可以避開 BOMs 對檔案進行修改及寫入的動作。 "UltraEdit" (試用軟件, 採取停用 BOM 寫入動作的選項) 及  "Notepad2 "(免費軟件, 選擇毋須簽署的方式進行 UTF-8 的編碼). 在此下載 Notepad2 : http://www.flos-freeware.ch/notepad2.html.

啟用問卷

當您滿意問卷的結構，您就可以 啟用 它。啟用 問卷會產生以下變化.

  1. 它建立一個 獨立的 MySQL 資料表去存放全部問卷結果，在資料表各欄位就會存放各可供選擇的答案.

  2. 它讓人可以輸入數據到該資料表，並讓您可以使用其他問卷功能，包括瀏覽及答卷代碼等。

  3. 它讓您可以使用 "答卷代碼" 功能。問卷一經啟用， "答卷代碼" 按鈕就可供使用了 (詳見"答卷代碼"的章節).

  4. 如果您的問卷被設定成 "非匿名", 系統就會馬上自動建立答卷代碼表。

在您 啟用問卷判，您要注意以下幾點:

  • 問卷一經建立，您仍可以更改題目的內容、答案及問卷等等，但就不能改變題型或答案類型.

  • 您不能夠新增題目或刪除題目，您亦不能新增答案到任何陣列或多項選擇題, 但是您可以新增答案到基本列表型的題目.

click 選 問卷按鈕選單的按鈕，就可以啟用問卷。如果本圖標未有顯示, 這表示您的問卷仍未具備可以啟用的狀態. 只要 click 選這個圖標， LimeSurvey 就會執行快速的全面檢查，以確保問卷可以運作正常.

安全地執行問卷

問卷一經啟用及有人填寫問卷後，以下有幾點值得注意:

  • 馬上輸出問卷結構及儲存 sql 檔案

  • 定期到訪 "瀏覽" 畫面及使用 "備份 問卷 x 個問卷結果" ，就可以建立一個 sql 的問卷結果資料表的備份檔.

  • 定期輸出問卷結果及保存一份檔案副本

  • 預防勝於治療，不要隨便改變問卷。 問卷啟用後部份問卷設定仍可以作出改變。在啟用問卷前，記緊要徹底測試您的問卷，之後就儘可能不要作出更動了。

  • 自從在 release 0.98finalRC1 的版本開始, 您可以替整個 LimeSurvey 數據庫備份(包括全部問卷、問卷結果、答卷代碼等)，方法是 click 選  管理按鈕選單的備份圖標。這樣就會輸出成一個 sql 檔案，日後您只要重新輸入這個檔案，就可以還原來的LimeSurvey 數據庫.

  • 如果您使用 phpMyAdmin 之類的設施, 就要定期備份整個 LimeSurvey 數據庫 (結構和數據)。遇有不幸發生整個數據庫遭毀滅的事件發生，您就可以重建數據庫及輸入備份資料表，這樣 LimeSurvey 就可以把破損的數據庫恢復原貌。

答卷代碼

在許多情況下您會邀請一群人士填寫問卷, 追跡有哪些人士 已完成問卷, 並確保每位人士都可以填寫問卷一次. 只要有問卷被啟用，您就可以建立答卷代碼. 答卷代碼功能有以下功能:

  • 輸入答卷人的姓名及電郵地址的名單 (以 CSV 檔案方式輸入) 或新增/移除個別答卷人士

  • 為每位答卷人士產生獨有的答卷代碼

  • 送出電郵邀請信給名單上的每位人士, 以集體信或個別電郵發出

  • 向名單上尚未回答問卷的人士發出提示信的電郵, 形式可以是集體電郵或個別電郵

  • 查看名單上誰人已回答問卷

  • 禁止未有問卷代碼的人士及已回答問卷的人士填寫問卷.

  • 編修/更改名單上的任何細節

當您一啟動問卷的答卷代碼功能, 就只有擁有答卷代碼的人士就可以填寫問卷. 換言之，只有能輸入正確的答卷代碼的人士才可以填寫公開的問卷.

答卷代碼的功能是配合所有 "非匿名"的問卷使用. 如果問卷是非匿名方式建立 (或 '跟進') 答卷代碼表就可以用來找出個別用戶對問卷作出的特定回應結果. 如果問卷是匿名方式建立，答卷代碼資料表與回應結果就變得毫無關連了.

下列是答卷代碼螢幕顯示的扼要功能選項:

  • 管理員: 返回管理問卷主畫面;

  • 綱要: 顯示答卷代碼表的扼要說明 (答卷代碼的數目、有多少已送出的邀請信、有多少已填寫問卷). 亦提供存取 '數據庫管理功能' (見下所示). 這是預設畫面;

  • 瀏覽: 顯示答卷代碼表的完整答卷人的名單. 在瀏覽螢幕您可以編修或刪除答卷代碼表的個別項目，亦可執行特定功能(詳見下列'瀏覽'章節)

  • 新增: 讓您為答卷代碼資料表新增項目;

  • 輸入: 讓您透過　csv 檔案輸入資料

  • 邀請信: 對於未受邀請填寫問卷但已擁有答卷代碼的人士，本功能會以集體電郵的形式邀請他們填寫問卷.

  • 提示信: 對於擁有答卷代碼的人士在收到第一次邀請信但尚未填寫問卷，本功能會以集體電郵的形式送出提示信給此等人士.

  • tokenify: 這是英語新詞, 源於本系統 (come on OED, get to work). 為全部答卷代碼資料表的項目建立獨有的答卷代碼.

本功能並非針對問卷的受訪者設計. 當初 LimeSurvey 被開發時, 它是用作傳統紙張式問卷的數據輸入而設計的. 數據輸入螢幕是用作收回問卷作大量數據輸入而設計的，稍後發展到讓鍵盤輸入問卷結果的用途 (使用跳位鍵 ,及鍵盤輸入).

輸出問卷結果

您可以透過瀏覽螢幕選擇 "輸出數據"　來輸出問卷結果.

當您選用這項功能, 您會看到一個新視窗, 列出一系列輸出的選項.

  • 縮寫標題字: 只輸出題目的id 及問卷 id

  • 完整的題目標題字: 把題目標題輸出為標題字
     

  • 短答: 只輸出每個答案的相關代碼

  • 延伸答案: 輸出吻合代碼的文本式答案
     

  • Microsoft Word 格式: 輸出數據為 '文件' (在此未被格式化的階段，數據僅由空格字符分隔)

  • Microsoft Excel 格式: 輸出數據為 '試算表'

  • CSV 逗號分隔的格式: 把數據輸出為逗號分隔的檔案

輸出螢幕讓您選定哪些欄位要被輸出；如果問卷並非匿名形式進行，您亦可以把每份回應問卷的答卷代碼合併輸出.

VVExport and VVImport

只要您開始收集問卷結果, 就就能夠依照問卷結果的表格直接進行編修數據，當中您會發現試算表工具十分有用.

VVExport and VVImport ( "VV" 代表垂直確認) 讓您可以把問卷結果的表格輸出成 "定位格分隔數值" (tsv) 檔案. 只要檔案結構亳無損，您甚至可以編修檔案內的數據值, 並使用 "VVImport" 功能輸入該格式的檔案，成為問卷結果的表格.

輸出 VV 問卷檔案
當您選擇使用 VVExport 來輸出問卷結果時，您會有以下選項︰輸出結果及保留問卷的啟用狀態, 或輸出結果及停用問卷.

如果您選擇停用問卷, 問卷結果的表格會被更改名稱， 但只限直接支援 MySQL 才有此功能 (正如停用問卷的情況一樣會被更改名稱).

如果您的問卷有使用答卷代碼，您必須停用問卷，但這樣答卷代碼的表格就會被更改名稱，而且不能再被使用. 因此，除非您懂得在 MySQL 更改資料表的名稱及懂得把代碼資料表用手動方式還原，否則您最好不要把使用答卷代碼的問卷停用.

懂得停用問卷十分重要，因為當題目／答案的組合有所更改， 'vv 問卷檔案' 讓您可以把回應結果輸入問卷. 如此能有效使到您可以在一個已啟用的問卷新增／移除題目.

Click 選 "輸出結果" 會產生一個可透過 Excel 或類似風格的試算表程式打開空格式分隔的檔案. 試算表的首兩列是欄位標籤. 首列是 "淺白易明" 的欄位，內容是題目／答案欄位的名稱. 第二列是有關問卷結果的內部 PHP 欄位名稱.

如果您想稍後把數據輸入到 LimeSurvey, 您就不應該修改本試算表的首兩列資料.

輸入 VV 問卷檔案
只要首兩列資料保持完好, 您就可以把 vv 問卷檔案的數據輸入到已啟用的 LimeSurvey 問卷內. 如果您輸出數據時已停用問卷，您就需要再用一貫的程序停用問卷，才可用執行本功能.

然後在瀏覽螢幕, 選擇 "輸入 VV 問卷檔案" 的圖標.

在 "輸入 VV 問卷檔案" 螢幕，您需要選擇 "vvexport" . 如果您選擇 "不排除紀錄 ID" 紀錄會被輸入及產生新的紀錄 ID. 如果您在方格取消 tick 選, 原來的紀錄 id 亦會一併被輸入.

有一系列選擇決定 LimeSurvey 如何處理雙項或多項的欄位資料.
 

統計

在瀏覽螢幕的選單上可以使用統計功能, 當然只有在問卷被啟用後才能使用本功能.

甚麼是統計功能?
可以這樣說，當您啟用了一個問卷，在它完成填寫問卷工作後，您打算開始分析它的問卷結果。LimeSurvey 可以把全部結果轉換成您需要的 Excel 試算表 ，或者是 CSV 檔案，又或者是 Microsoft Word 檔案, 這樣您就可以隨心所欲地運用您的問卷數據了。又或者您可以運用統計功能來檢視數據, 享受這些有用數據帶給您的各種效益吧！

統計功能使您可以'篩選' 數據、統計數字及顯示個別欄位的摘要. 統計程序未必能會執行高難度的篩選工序 - 但您會對系統能輕而易舉地把資訊排序列出感到十分有意思.

篩選數據

當您首先使用統計功能的時候，系統會列出一系列問題用來幫助您篩選數據. 舉例如下︰ 每道題目都要輸入題目編號， '小圓' 按鈕用來顯示本表格的摘要, 並出現喇叭的圖標，click 選此圖標後，會有這道題目的相關文字說明。 它會為該道題目列出一系列預先定義好的答案. 該列表容許您多重選擇. 因此，在以下的例子，您可以在選定 "進展中"的過程，篩選全部輸出結果，又或者改用控制按鈕，您可以新增 "固定條款" 或 "自定條款". 透過所有過濾框的設定可以篩選出合您意的輸出結果.
